This project analyses multi-temporal land-use and land-cover (LULC) change in Bhubaneswar, Odisha using freely available Sentinel-2 Surface Reflectance imagery processed through Google Earth Engine. Four consecutive two-year periods from 2018 to 2026 are compared to track urbanisation, vegetation change and water-body dynamics across the Bhubaneswar Municipal Corporation area.

All processing happens server-side on a secure Render.com backend; no Google Earth Engine account is needed to use the interactive map.

Methodology

1. Image Compositing

For each year, a February–March median composite is generated from the Sentinel-2 SR Harmonized collection (COPERNICUS/S2_SR_HARMONIZED). Using a consistent phenological window (peak Rabi / winter-crop season) across all years eliminates inter-year spectral drift caused by seasonal NDVI variation — a common source of false change detections.

# Feb–Mar composite (Sentinel-2, cloud-masked, median) img = (ee.ImageCollection("COPERNICUS/S2_SR_HARMONIZED") .filterDate(f"{year}-02-01", f"{year}-03-31") .filterBounds(roi) .map(_mask_s2_clouds) .median() .select(["B3","B4","B8","B11"]) .divide(10000) .clip(roi))

2. Spectral Indices

Three normalised spectral indices are computed per composite:

IndexFormulaInterpretation
NDVI(NIR − Red) / (NIR + Red)Vegetation density
NDWI(Green − NIR) / (Green + NIR)Open water bodies
NDBI(SWIR1 − NIR) / (SWIR1 + NIR)Built-up surfaces
BUINDBI − NDVIBuilt-up vs vegetation discrimination

The Built-up Index (BUI = NDBI − NDVI) is key: during the Rabi crop season, agricultural fields have high NDVI (BUI ≪ 0), while concrete and impervious surfaces maintain high NDBI and low NDVI (BUI > 0). This dramatically reduces the false built-up signal from bare harvested fields.

3. Land-Cover Classification

water = ndwi.gt(0.10) vegetation = ndvi.gt(0.30).And(ndwi.lte(0.10)) builtup = bui.gt(0.0).And(ndwi.lte(0.05))

Each pixel is assigned to one of three primary classes. Pixels that meet no threshold (bare soil, cloud shadow remnants, mixed pixels) are left unclassified.

4. Change Detection

Six change classes are derived by comparing the classified maps of year₁ and year₂:

ClassConditionColour
Built-up GainNon-built → Built (year₁ → year₂)■ Red
Built-up LossBuilt → Non-built■ Orange
Vegetation LossVeg → Non-veg■ Yellow
Vegetation GainNon-veg → Veg■ Green
Water GainNon-water → Water■ Blue
Water RecessionWater → Non-water■ Cyan

5. Area Statistics

Areas are computed using ee.Image.pixelArea() (geodetically correct m² per pixel) multiplied by the class mask and summed via ee.Reducer.sum() at 60 m resolution. This avoids the fixed-scale assumption error where bestEffort=True silently coarsens the pixel size, making year-to-year comparisons unreliable.
